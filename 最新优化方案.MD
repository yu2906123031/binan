## 最新实施方案

### 第一阶段：profile 与方向开关接入

目标：先把 aggressive 模式以最小侵入方式挂进现有入口。

实施项：
1. 新增三个 profile JSON
   - `profiles/aggressive-micro-scalp-long-short.json`
   - `profiles/aggressive-micro-scalp-long-only.json`
   - `profiles/aggressive-micro-scalp-short-only.json`
2. 在 `apply_runtime_profile()` 注册三个 profile 名称
3. 增加 `--enabled-sides`
4. 新增：
   - `normalize_enabled_sides(value)`
   - `side_is_enabled(args, side)`
5. 统一别名：
   - `both` → `long,short`
   - `all` → `long,short`
   - `buy` → `long`
   - `sell` → `short`
6. CLI 显式参数优先级高于 profile
7. 扫描阶段对 disabled side 直接跳过

完成定义：
- long-only 模式下 short 完全不参与评分、trigger、下单
- short-only 模式下 long 完全不参与评分、trigger、下单
- funnel 输出新增：
  - `aggressive_mode`
  - `scan_interval_sec`
  - `monitor_interval_sec`
  - `enabled_sides`
  - `skipped_disabled_side_count`
  - `trigger_fast_path_count`
  - `micro_scalp_candidates_count`

### 第二阶段：aggressive 扫描与入场快路径

目标：提高扫描频率与进场速度，同时保留硬 veto。

实施项：
1. aggressive profile 默认：
   - `poll_interval_sec = 15`
   - `monitor_poll_interval_sec = 3`
   - `max_scan_cycles = 0`
   - `max_open_positions = 1`
2. 降低 setup 与 trigger 门槛：
   - `min_5m_change_pct = 0.25`
   - `min_volume_multiple = 0.9`
   - `trigger_min_confirmations = 1`
3. 增加 aggressive fast path 标记
4. 保留以下硬拒绝：
   - 深度不足
   - 滑点过高
   - OI 反向
   - CVD 背离
   - funding 过高
   - 过热追价
   - BTC/SOL 市场环境硬风险过滤

完成定义：
- aggressive 允许更早进入 `setup_ready` / `trigger_fired`
- aggressive 不绕过 `apply_hard_veto_filters()` 一类的硬风控链路

### 第三阶段：micro-scalp 出场管理

目标：高胜率优先，快进快出，小 R 连续积累。

实施项：
1. aggressive profile 默认：
   - `tp1_r = 0.45`
   - `tp1_close_pct = 0.65`
   - `tp2_r = 0.8`
   - `tp2_close_pct = 0.25`
   - 剩余 runner 10%
   - `breakeven_r = 0.35`
2. 新增参数：
   - `--micro-scalp-mode`
   - `--micro-scalp-time-stop-sec`
   - `--micro-scalp-min-profit-r`
   - `--micro-scalp-max-hold-sec`
3. 新增逻辑：
   - 180 秒未达到 `0.15R`，触发 `time_stop`
   - 600 秒仍未达到 TP2，触发 `force_exit` 或 trailing 收口
4. 新增事件：
   - `micro_scalp_time_stop`
   - `micro_scalp_breakeven_move`

   - `micro_scalp_force_exit`

### 第五阶段：actual risk 统计修复

目标：让小资金 micro-scalp 的风险统计真实反映被 notional 截断后的实际风险。

实施项：
1. Candidate 增加：
   - `intended_risk_usdt`
   - `actual_risk_usdt`
   - `risk_scale_ratio`
   - `position_size_capped_by_notional`
2. 统一计算：
   - `intended_risk_usdt = risk_usdt`
   - `actual_risk_usdt = abs(quantity * risk_per_unit)`
   - `risk_scale_ratio = actual_risk_usdt / intended_risk_usdt`
   - `position_size_capped_by_notional = actual_risk_usdt < intended_risk_usdt * 0.98`
3. position payload、event payload、trade journal 全量落盘

完成定义：
- 所有 live 开仓事件都能看见理论风险与实际风险差值
- 后续统计可以区分“风控设定风险”和“实际成交风险”
