# 当前下单策略说明

更新时间：2026-04-26

## 当前运行模式

当前策略运行在 OKX 模拟盘模式，不是 Binance 实盘。

当前后台命令：

```powershell
python main.py --profile okx-sim-active --live --okx-simulated-trading --auto-loop --max-scan-cycles 0 --poll-interval-sec 60 --output-format json
```

关键含义：

- `--profile okx-sim-active`：使用 OKX 模拟盘活跃档。
- `--live`：允许进入下单执行流程。这里必须开启，否则只扫描不执行。
- `--okx-simulated-trading`：把执行目标切到 OKX 模拟盘，不走 Binance 实盘。
- `--auto-loop`：持续循环扫描。
- `--max-scan-cycles 0`：无限循环。
- `--poll-interval-sec 60`：每 60 秒扫描一轮。

## 下单流程

策略不是看到涨跌就直接下单，而是按下面流程逐层过滤：

1. 生成候选池

   来源包括热门币、24h 涨幅榜、24h 跌幅榜，以及手动指定交易对。每个交易对会同时评估做多和做空两个方向。

2. 早期过滤

   先排除明显不适合交易的标的，例如：

   - 做多突破未确认
   - 做空破位未确认
   - 近 5 分钟波动不足
   - 成交额不足
   - 止损距离无效
   - 大周期方向不支持
   - 资金费率不合适

3. 候选打分

   通过早期过滤后，策略会计算候选评分。评分参考：

   - 5 分钟价格动量
   - 成交量放大倍数
   - 15 分钟加速度
   - 1 小时 / 4 小时趋势
   - RSI
   - EMA20 / VWAP 距离
   - 资金费率
   - OI 持仓量变化
   - CVD 主动买卖流
   - taker 买卖比例
   - 盘口流动性和预估滑点

4. 结构确认

   候选会进入两阶段确认：

   - `setup_ready`：结构已经准备好，可以继续观察。
   - `trigger_fired`：触发条件已经出现，可以进入下单流程。

   只有候选同时满足结构和触发，才会允许下单。

5. 风控检查

   下单前还会做最后风控：

   - 策略是否被暂停
   - 是否达到日亏损限制
   - 是否达到连续亏损限制
   - 当前币种是否处于冷却期
   - 是否已有持仓
   - 多空持仓数量限制
   - 净暴露 / 总暴露限制
   - 单币种同向 / 反向限制
   - 组合热度限制
   - 滑点和流动性风险
   - CVD 背离
   - OI 反转
   - 派发风险

6. OKX 模拟盘下单

   只有所有检查通过后，才会调用 OKX 模拟盘下单接口。

## 当前活跃档参数

当前使用 `okx-sim-active`，这是为了模拟盘验证而优化的活跃档。

核心参数：

| 参数 | 当前值 | 含义 |
| --- | ---: | --- |
| `risk_usdt` | 1.0 | 单笔基础风险约 1 USDT |
| `max_notional_usdt` | 300 | 单笔最大名义仓位 |
| `leverage` | 3 | 默认杠杆 |
| `lookback_bars` | 3 | 突破判断回看 3 根 5m K 线 |
| `swing_bars` | 4 | 止损结构回看 4 根 5m K 线 |
| `min_quote_volume` | 5,000,000 | 24h 最低成交额 |
| `top_gainers` | 40 | 扫描涨幅榜前 40 |
| `top_losers` | 40 | 扫描跌幅榜前 40 |
| `max_candidates` | 12 | 最多保留 12 个候选 |
| `min_5m_change_pct` | 0.5 | 最低 5m 方向波动 |
| `min_volume_multiple` | 1.05 | 最低放量倍数 |
| `setup_breakout_tolerance_pct` | 0.8 | 距离突破 0.8% 内允许进入候选观察 |
| `max_rsi_5m` | 84 | 做多 RSI 上限 |
| `max_funding_rate` | 0.0008 | 资金费率上限 |
| `max_funding_rate_avg` | 0.0005 | 平均资金费率上限 |

## 为什么现在能看到候选但还不下单

优化前，很多标的在早期过滤阶段就被丢弃，界面经常显示 `候选数: 0`。

现在 `okx-sim-active` 允许“接近突破”的标的进入候选池，所以可以看到候选数，例如 `候选数: 7`。

但进入候选不等于马上下单。比如界面显示：

- 结构未就绪
- 触发未出现
- 等待真正突破
- 回踩支撑未确认
- 弹性回踩未确认
- 存在追高 / 过热风险

这表示策略已经发现了值得观察的机会，但还没有达到实际下单条件。

## 下单必须满足的关键条件

一个候选要进入 OKX 模拟盘下单，至少需要：

1. `setup_ready = true`
2. `trigger_fired = true`
3. 风控状态为允许
4. 没有硬拒绝原因
5. 没有明显滑点风险
6. 没有 CVD 背离
7. 没有 OI 反转
8. 没有派发风险
9. 持仓数量和暴露限制允许

如果界面显示 `candidate_setup_not_ready`，意思是候选结构还没准备好。

如果界面显示 `candidate_trigger_not_fired`，意思是结构可以看，但触发条件还没出现。

## 界面字段解释

### 运行模式

显示当前策略是在扫描、待机、Binance 实盘，还是 OKX 模拟盘。

当前应显示：`OKX 模拟盘`。

### 交易动作

显示当前是否已经触发下单。

- `未下单`：本轮没有进入执行。
- `等待执行`：有候选，但还没通过触发或风控。
- `模拟盘已下单`：已经向 OKX 模拟盘提交订单。

### 市场状态

根据 BTC 和 SOL 的趋势、动量判断整体环境。

- `偏强`：市场支持更积极做多。
- `中性`：市场没有强烈方向。
- `谨慎`：部分环境不支持。
- `偏弱`：风险较高，仓位和评分会受影响。

### 风控状态

显示当前是否允许下单。

- `允许`：没有风控拦截。
- `拦截`：候选被风控阻止，需要看具体原因。

### 选中标的

显示当前分数最高的候选。

重点看：

- `状态`：候选属于启动形态、蓄势、过热等。
- `评分`：综合评分。
- `执行结论`：当前为什么能下单或不能下单。
- `关键触发`：结构和触发是否满足。
- `突破距离`：距离突破位置还有多远。
- `止损距离`：入场到止损的大致风险距离。
- `成交质量`：盘口深度和滑点质量。
- `未满足项`：当前卡住的具体条件。

## 常见不下单原因

| 界面原因 | 含义 |
| --- | --- |
| 候选结构未就绪 | 有候选，但结构还没达到可交易状态 |
| 候选触发未出现 | 结构可以观察，但还没有触发下单信号 |
| 做多突破未确认 | 价格还没真正突破近期高点 |
| 做空破位未确认 | 价格还没真正跌破近期低点 |
| 近 5 分钟波动不足 | 当前动量不够 |
| 放量不足 | 成交量没有明显放大 |
| 大周期方向不支持 | 1h/4h 趋势不支持该方向 |
| 持仓量反转否决 | OI 变化不支持当前方向 |
| 滑点风险否决 | 盘口执行质量不够 |
| 价格延伸过远 | 有追高或追空风险 |
| CVD 背离 | 主动买卖流和价格方向不一致 |

## 对账说明

OKX 模拟盘模式下，主循环会跳过 Binance 实盘对账，界面中看到的“对账通过”表示本地策略状态没有发现孤儿仓位或缺保护单。

独立运行 Binance 对账时使用：

```powershell
python main.py --reconcile-only --output-format json
```

如果 Binance 返回 `Timestamp for this request is outside of the recvWindow`，说明本机时间和交易所服务器时间偏差过大，需要先校准系统时间。

## 当前建议

当前已经从“完全没有候选”优化为“有候选但等待触发”。这是更合理的模拟盘状态。

下一步观察重点：

1. 候选是否能从“结构未就绪”变为“结构已就绪”。
2. 是否出现 `触发已出现`。
3. 是否被风控原因拦截。
4. 如果长时间只有候选但没有触发，再考虑继续放宽触发确认，而不是直接绕过风控。
